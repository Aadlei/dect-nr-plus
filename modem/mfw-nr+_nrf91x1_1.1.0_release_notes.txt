mfw-nr+_nrf91x1 release notes
-----------------------------

** Introduction to the mfw-nr+_nrf91x1 release notes **

These release notes describe the changes in the mfw-nr+_nrf91x1 firmware from version to version.
The release notes are intended to briefly list all relevant changes in each version.

*** mfw-nr+_nrf91x1_1.1.0
**********************************

The mfw-nr+_nrf91x1 is the DECT NR+ firmware for modem subsystem of nRF9131 LACA A0, nRF9151
LACA A0, nRF9151 LACA A1 and nRF9161 LACA A0 SiP modules. It supports Release 1 of the DECT-2020 New Radio (NR)
technical specifications version 1.5.1.

DECT NR+ operation is enabled on frequency bands 1, 2, 4, 9, and 22.

DECT NR+ band 4 is supported by the nRF9151-LACA-A0 and nRF9151-LACA-A1.

It is advisable to use the latest versions of the nRF Connect and nRF Connect Programmer tools.

This release aligns with v1.2 of the nRF91x1 DECT NR+ AT Commands Reference Guide.

*** New functionality
*********************
- Added support for adding and removing scheduled operations, enabling a 1-slot turnaround time for
  PHY operations in low-latency modes.
- Enhanced support for various radio modes to improve power consumption across different use cases.
- Added reporting of turnaround times for different radio modes.
- Introduced support for DECT NR+ band 4, available on the nRF9151 LACA A0 and nRF9151 LACA A1.
- Add support for 21 dBm output power on bands B1, B2, B9, and B22, available on the nRF9151 LACA A0 and A1.
  Note: Output power on Band 4 is limited to 19 dBm.
- Included battery voltage values in PHY operation callbacks.
- Added operation handle values for all PCC and PDC reception callbacks, simplifying packet
  identification for RX operations.

*** Bug fixes
*************
-

*** Limitations
***************
- Delta FOTA is not supported from previous mfw-nr+_nrf91x1_1.0.x releases

*** mfw-nr+_nrf91x1_1.0.1
*************************

The mfw-nr+_nrf91x1 is the DECT-2020 New Radio (NR) firmware for modem subsystem of nRF9131 LACA
A0, nRF9151 LACA A0, and nRF9161 LACA A0 SiP modules. It supports Release 1 of the DECT-2020 New
Radio (NR) technical specifications version 1.5.1.

DECT-2020 New Radio (NR) operation is enabled on frequency bands 1, 2, 9 and 22.

It is advisable to use the latest versions of the nRF Connect and nRF Connect Programmer tools.

This release aligns with v1.1 of the nRF91x1 DECT NR+ AT Commands Reference Guide.

*** New functionality
*********************
-

*** Changes
***********
- Updated physical layer firmware latencies, affecting values reported in the libmodem API
  nrf_modem_dect_phy_latency.
- Improved RX AGC loop for better gain control accuracy, enhancing RSSI-1 measurement accuracy.
- Improved AGC performance to mitigate adjacent channel interference.
- Performance improvements for decoding at low signal-to-noise ratios.
- Updated the sequence for indicating and tracing PHY coredump, allowing the application to capture
  PHY coredumps within the fault handler.

*** Bug fixes
*************
- Addressed a possible memory leak issue in combined TX and RX operations that occurred when the
  operation was scheduled but the start time expired before the actual start.
- Fixed the status return value during de-initialization when the temperature exceeds the maximum
  supported value.
- Fixed an issue with RX stopping during de-initialization if combined TX and RX operation was used.
  Previously, RX operation was not stopped if de-initialization was performed during TX operation.
- Adjusted the timing of as-soon-as-possible TX operations to prevent late starts.
- Made minor corrections and improvements to RSSI reports generated by the physical layer firmware.
- Resolved excessive current consumption after de-initializing the physical layer firmware.
- Resolved excessive current consumption with nRF9151 after TX RF off.

*** Limitations
***************
- Power consumption has not been fully optimized.

*** mfw-nr+_nrf91x1_1.0.0-4.prealpha
************************************

The mfw-nr+_nrf91x1 is the DECT-2020 New Radio (NR) firmware for the modem subsystem of nRF9161 and
nRF9131 devices. It supports Release 1 of the DECT-2020 New Radio (NR) technical specifications.

DECT-2020 New Radio (NR) operation is enabled on frequency band 1, 2, 9 and 22.

It is recommended to use the latest version of nRF Connect and nRF Connect Programmer tools.

This release version supports nRF9161 LACA A0 and nRF9131 LACA A0 SiP versions.

*** New functionality
*********************
- Added support for frequency band 22.
- Implemented support for combined RX and TX operations.

*** Changes
***********
- Enhanced TX filtering to mitigate adjacent channel interference.
- Removed deep sleep-related latency values and updated latency timings.
- Enhancements on RSSI-1 scan accuracy.

*** Bug fixes
*************
- Fixed an issue with redundant RSSI reports. Under certain PHY operation timings, an extra and
  unnecessary RSSI report was possible.
- Resolved an issue with the TX HARQ process count. Without the fix, the number of TX HARQ processes
  remained constantly at 2.

*** Limitations
***************
- This release version is intended for R&D and test purposes only, and should not be used in
  products or for deriving products from this firmware.
- Performance, such as firmware operation in dispersive radio signal conditions, has not been
  optimized.
- Dynamic sleep mode is not supported, affecting overall power management.

*** mfw-nr+_nrf91x1_0.5.0-74.prealpha
*************************************

The mfw-nr+_nrf91x1 is the DECT-2020 New Radio (NR) firmware for the modem subsystem of nRF9161 and
nRF9131 devices. It supports Release 1 of the DECT-2020 New Radio (NR) technical specifications.

DECT-2020 New Radio (NR) operation is enabled on frequency band 1, 2 and 9.

It is recommended to use the latest version of nRF Connect and nRF Connect Programmer tools.

This release version supports nRF9161 LACA A0 engineering and nRF9131 LACA A0 engineering SiP
versions.

*** New functionality
*********************
- HARQ soft bit combining. New data indication bit has effect in physical layer control header
  type 2.
- Modem subsystem thermal protection. Radio operations are prohibited if temperature conditions
  exceed the operations limits set in nRF9131 and nRF9161 product specifications. Current
  temperature value is returned in operation callbacks.
- Automatic mechanism to lock nRF9131 calibration data. Automatic lock is performed when modem is
  first time initialized with "nrf_modem_init". If nRF9131 calibration is not performed and modem
  initialization is executed, error code is returned.
- Continuous Wave (CW) support for unmodulated TX RF measurements.

*** Changes
***********
- Updated physical layer latencies that affects libmodem API "nrf_modem_dect_phy_latency".

*** Bug fixes
*************

*** Limitations
***************

  or deriving products from this firmware.
- Support of HARQ is limited.
- Power consumption has not been optimized.
- There are some L1 operation latency restrictions as reported by the modem.
- Performance, for example firmware operation in weak radio signal conditions or condition with
  control or data channel retransmissions, has not been optimized.
